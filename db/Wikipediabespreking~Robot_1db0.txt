[wysig] Zin en onzin van robots

   Robots zijn een goed hulpmiddel om werk uit te voeren waar weinig
   denkwerk voor nodig is. Het gevaar van robots is dat ze niet kunnen
   denken en dus fouten maken.

   Mijn ideeën over robots zijn als volgt:
     * Een robot moet altijd handmatig bediend worden. Autonome robots
       zijn ongewenst.
          + Reden: Een autonome robot kan niet denken en maakt fouten.
     * Een robot moet vertrouwd worden. Een robot heeft een meester nodig
       om hem te bedienen, de meester moet ook vertrouwd zijn.
          + Reden: De werking van de robot moet gecontroleerd worden door
            zijn meester, indien iemand een robot verkeerd bediend
            ontstaan fouten.
     * Een robot mag maximaal één taak per rekening uitvoeren. Een
       interwikibot gebruikt dus een andere rekening dan een
       speltoetsrobot, ook al worden beiden door dezelfde meester bediend.
          + Reden: Het feit dat één robot goed werkt betekent niet dat de
            andere goed werkt.

   Een robot die een robotvlaggetje wil krijgen, moet aan alle drie
   voorwaarden hierboven voldoen, alsook:
     * Er bestaat geen reden om de wijziging in de onlangse wijzigingen te
       zien.

   Een interwikirobot kan ongezien zijn werk doen. Een speltoetsrobot wat
   mij betreft niet, er is geen reden waarom dit onzichtbaar moet
   gebeuren. Danielm 16:24, 28 Augustus 2006 (UTC)

                'n idee van die voor- en nadele van robotte te gee. Wat
                vir my baie sin maak is die volgende deel: Bot operators

                interwiki linkings between the two encyclopedias are done
                correctly.

                2. The period, if any, we should expect it to run
                3. What language or program it is running
                4. Its purpose

                Dit is 'n goeie idee om dan miskien 'n proeftydperk toe te
                ken, waarin die bot se gedrag beoordeel kan word. Soos

[wysig] About the advantage of having many interwiki bots

   You should not compare the number of interwiki bots to the local number
   of articles and users but to the overall number of articles in all
   Wikipedias. The bots follows interwiki links from all languages to all
   languages, and after all referenced pages have been collected, they
   will update those with missing or wrong links. For that process to be
   efficient it is best if the bots can edit in as many languages as
   possible.

   Besides there are differences in the way the bots work. Where they
   start, how the articles are selected, and whether they run
   automatically (and thus may have to skip when more than one link is
   found to the same language) or as mine run manually assisted so the
   operator may (or may not) be able to solve the found interwiki
   conflicts.

   As I see it, you will by allowing use of interwiki bots improve both
   your own Wikipedia by quicker update of interwiki links, and all
   Wikipedias by more efficient use of the resources - specially if a bot
   is able to solve an interwiki conflict so the next bot can save many
   fetches of wrong pages. Byrial 10:34, 29 Junie 2007 (UTC)

          Especially for the smaller Wikipedias, we observe that editors
          very often set outgoing interwiki links but fail to add incoming
          ones to foreign wikipedias. Thus, unless a bot starts from their
          wiki, these interwiki links are never detected and never get
          propagated anywhere. In other words, we need interwiki bots
          starting from every Wikipedia. --Purodha 13:31, 4 Februarie 2008
          (UTC)
